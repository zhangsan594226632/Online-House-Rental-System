## 一、项目概述

### 1\.1 项目背景

随着城市流动人口持续增长，租房需求逐年提升，传统线下租房、中介租房模式存在信息不透明、房源管控难、交易流程繁琐、沟通渠道单一等痛点。为解决租客找房难、房东房源管理麻烦、平台管理员无法统一监管房源租赁全流程的问题，开发一套前后端分离的租房管理系统，实现租客在线看房、资讯查阅、租房攻略发布；房东发布房源、管理租赁订单；管理员后台统一管控用户、房源、租赁、预约数据并可视化统计分析。

### 1\.2 开发技术栈

#### 后端技术（SpringBoot）

- 核心框架：SpringBoot 2\.7\.x，快速搭建 Java 后端服务

- 持久层：MyBatis/MyBatis\-Plus，简化数据库 CRUD 操作

- 数据库：MySQL 8\.0，存储房源、用户、租赁、资讯、攻略等业务数据

- 安全认证：JWT Token 实现登录鉴权，区分管理员、房东、普通租客权限

- 工具组件：EasyExcel（合同导出）、POI、Hutool 工具类

- 图表支持：ECharts 后端数据接口，提供房源统计数据

- 文件上传：OSS / 本地文件存储，房源图片、资讯图片上传

#### 前端技术（Vue2/Vue3 \+ ElementUI）

- 核心框架：Vue3 \+ Vue Router \+ Vuex/Pinia 状态管理

- UI 组件库：Element Plus（后台管理端）、Element UI（用户前台）

- 图表可视化：ECharts 实现饼图、柱状图、折线图房源统计

- 富文本编辑器：wangEditor，用于租房攻略、资讯内容编辑

- 网络请求：Axios 封装请求拦截、响应拦截，统一处理 Token 过期

- 页面布局：响应式布局，适配 PC 端访问

### 1\.3 系统角色划分

1. **普通租客（前台用户）**：浏览首页房源、查看房屋资讯、筛选房源、阅读 / 发布租房攻略、个人中心管理我的发布、登录注册

2. **房东角色**：注册房东账号，发布房源、管理租赁订单、处理看房预约

3. **系统管理员（后台）**：管理全部用户（租客 \+ 房东）、房源信息、房屋租赁订单、看房预约、房屋地区 / 类型分类、可视化数据统计、租赁审核

### 1\.4 系统功能总览

系统分为**前台租客展示端**、**后台管理员管理端**两大模块，三大角色权限隔离。

### 2\.1 前台租客功能模块（用户首页）

#### 2\.1\.1 登录注册页面

- 角色切换登录：管理员、房东双标签登录入口

- 房东注册功能：租客可注册成为房东，填写账号密码完成注册

- 表单校验：账号、密码非空校验，登录成功携带 Token 跳转对应页面

#### 2\.1\.2 系统首页

1. 顶部导航栏：首页、房屋资讯、房源信息、攻略信息、个人中心

2. 轮播横幅：城市房产背景轮播图

3. 房源信息展示区：卡片式展示房源缩略图、房屋名称、地址、月租价格，提供「全部」跳转完整房源列表

4. 房屋资讯展示区：租房行业资讯卡片，提供全部资讯入口

#### 2\.1\.3 房源信息页面

- 多条件筛选：房屋名称关键词、月租价格区间（最小 / 最大）、地区下拉筛选

- 房屋类型标签快速筛选：全部、2 室 1 厅、各类自定义房型

- 卡片分页展示房源：房源实景图、房屋名称、月租，点击卡片查看房源详情

#### 2\.1\.4 房屋资讯页面

- 顶部轮播图，资讯标题搜索框

- 返回按钮：回到系统首页

房型字典表：2 室 1 厅、一室一厅等房型名称，用于前台筛选、后台统计。

### 3\.7 area 地区表

房屋所属地区分类，用于房源地区筛选。

## 四、前后端核心业务流程

### 4\.4 用户登录鉴权流程

1. 前端输入账号密码，调用后端登录接口

2. 后端校验账号密码，验证角色（管理员 / 房东 / 租客）

3. 验证通过生成 JWT Token 返回前端

4. 前端将 Token 存储 localStorage，Axios 请求头自动携带 Token

5. 后端拦截器校验 Token 有效性、过期时间，无 Token / 失效则跳转登录页

### 4\.2 租客租房完整流程

1. 前台浏览首页 / 房源列表，筛选心仪房源

2. 查看房源详情，发起预约看房

3. 与房东确认后提交租赁订单，选择租赁时长

4. 管理员后台审核租赁订单，填写审核回复

5. 审核通过后租客完成租金支付，系统生成租房合同

6. 后台管理员可下载、归档租赁合同，查看全部租赁统计数据

### 4\.3 攻略发布流程

1. 租客进入攻略模块，点击发布新帖

2. 填写标题、选择公开 / 私人，富文本编辑器编写租房经验

3. 提交保存至 rent\_strategy 表，个人中心「我的发布」可管理帖子

4. 全部公开攻略在前台攻略页面展示，所有用户可浏览

### 4\.4 管理员数据统计流程

1. 后台房源管理页点击「类型统计」按钮

2. 前端调用后端统计接口，分组查询各房型 / 价格 / 面积房源数量

3. 后端返回统计数组，ECharts 渲染饼图、柱状图可视化弹窗展示

## 五、项目部署说明

### 5\.1 后端部署步骤

1. MySQL 导入项目 sql 脚本，创建对应数据库、数据表

2. 修改 application\.yml 配置文件：数据库地址、账号密码、文件上传路径、JWT 密钥

3. Maven 打包 SpringBoot 项目：`clean package -DskipTests` 生成 jar 包

4. 服务器执行命令启动：`java -jar rent-system.jar --port=8080`

### 5\.2 前端部署步骤

1. 进入 Vue 前端项目目录，安装依赖：`npm install`

2. 打包生产文件：`npm run build`，生成 dist 文件夹

3. Nginx 配置静态资源托管 dist，反向代理后端 8080 接口地址

4. 访问 Nginx 端口即可打开租房系统前台、后台管理页面

## 六、系统特色与创新点

1. **多角色权限分离**：租客、房东、管理员三套权限体系，页面、接口数据隔离，越权访问拦截

2. **可视化数据大屏**：后台 ECharts 多维度房源统计，直观展示房型、价格、面积分布

3. **富文本租房攻略**：支持图文混排发布租房经验，搭建租客交流社区

4. **多维度房源筛选**：名称、价格区间、地区、房型标签组合筛选，快速定位房源

5. **完整租赁业务闭环**：预约看房 \- 租赁下单 \- 管理员审核 \- 租金支付 \- 合同导出完整租房流程

6. **前后端完全分离**：SpringBoot 提供 RESTful 接口，Vue 独立前端，便于迭代、二次开发

## 七、项目总结

本基于 SpringBoot\+Vue 的租房管理系统，覆盖租房行业前台用户看房、资讯交流、攻略分享需求，同时提供完整后台管理功能实现房源、租赁订单、用户统一管控，搭配可视化数据分析模块降低管理员运营成本。系统架构清晰、功能完整、业务贴合真实租房场景，可作为毕业设计、中小型租房平台商用基础版本，可扩展功能包含在线聊天、房源收藏、租金分期、地图找房等模块。
